# カレンダーアプリ

## アプリ概要

このアプリは日付の表示や予定の追加､編集､削除などの一般的なカレンダーアプリを模したものです｡

主にReactやFirebaseといったフロントエンドの技術を使用して作成しております｡

このアプリでは認証機能などの実装は行っていませんので､以下のurlからすぐ触って挙動を確かめることが可能です｡

<https://sakanobu-calendar.firebaseapp.com>

## アプリケーションの機能一覧

- カレンダー表示
- 予定の追加(タイトル､メモ､時間､タグ)
- 予定の編集および削除
- タグによる一部の予定の表示および非表示の取捨選択
- 他の年月のカレンダーからの本日のカレンダーへと一気にジャンプ可能
- 指定した年月のカレンダーへと一気にジャンプ可能

## カレンダーアプリ作成の動機

IT業界未経験者の就活体験記のように作りたいアプリのアイディアや現実世界で抱える課題などがパッと思い浮かべたら良かったのですが､私にはピンと来るものがありませんでした｡

そこで､以下の2つの理由からフロントエンドの技術を中心にカレンダーアプリを作ることに決めました｡

1. 形あるモノの作成に長けているフロントエンドという技術領域の腕を磨きたい
1. フロントエンドの練習としてカレンダーアプリは丁度いいレベル感である

### 1について

プログラミングを学んでいて常々感じていたのは､｢人は文字や図がないと全く思考が進まない｣ということでした｡

ひとえに機能を実装するといってもそこには､要件定義に基づいた仕様の策定/DBのテーブル設計/UIの設計/アプリ全体の設計/コーディング…etc､と考慮しなければならないことが多岐にわたります｡

そのような中で頭だけで考えようとしてしまうと､曖昧な部分への懸念や場合分けで頭の中が膨れ上がりいつの間にか堂々巡りになってしまう…ということが私には何度もありました｡

加えて､実際の業務であればその問題は個人の問題に留まらず､チームのメンバーやITに疎い可能性のある顧客などにも広がるため､各々の認識のズレによって手戻りや意見の衝突が生まれてしまうのは容易に想像できてしまいます｡

そのような問題意識から｢人々の認識のすり合わせに向いているITの領域は何か?｣と考えたとき､やはりUIという視覚的に明らかなものを生業とするフロントエンドエンジニアが魅力的に映り､TypeScriptやReactを使ったアプリ開発を行おうと思い立ちました｡

### 2について

1で述べた理由から､数ヶ月前ToDoリストの作成に取り掛かっていたところ､以下の記事を見かけました｡

[2020年にWebフロントエンドを勉強する人が作るべきたったひとつのアプリ - Qiita](https://qiita.com/Dragon-taro/items/03f322dee15b19c33613)

この記事の本文にあるようにカレンダーはコンポーネントやStateの量が程よく存在し､かつ､ToDoリストと異なり日付系のライブラリにもしっかり着手しなければならないので丁度いい題材だなと思いました｡

冒頭で述べた通り､私には作り出したいオリジナルアプリは特に無く､フロントエンドの練習になるような経験を積みたいという気持ちも強かったので､ムリにアイディアをひねり出そうとせずカレンダーアプリの実装による鍛錬をしてしまおうと考えました｡

ちなみに､本来の業務では推奨されないはずですが､経験を積みたい技術ありきで技術選定などを進めていったため､TypeScriptを用いたReactの開発//Custom Hooksの練習/FirebaseというBaaSを用いた開発/ESLint, Prettier, Jestなどの環境構築…etcなど､幅広く経験をすることが出来たのは良かったと思います｡

### なぜReactを使ったのか?

素のJavaScriptやTypeScriptでフロントエンドを実装していくのは辛いのため､QiitaやZennなどで今も盛んに記事が投稿されているReactとVue.jsのどちらか一方を選ぼうと当初から半ば決めつけていました｡
両者の公式サイトや紹介記事を読む中で､ちょうど技術書典9で全3巻に及ぶReactの技術同人誌である『[りあクト！ TypeScriptで始めるつらくないReact開発](https://techbookfest.org/product/5436045076201472?productVariantID=6240785991204864)』が出版されると聞いて飛びついてしまい､そのままの流れでアプリ開発もReactにしてしまいました｡
今振り返ってみるととても安直な決め方であったなと思いますが､とりあえず学んで触ってみないことには比較もできず足踏みするばかりなので結果的には仕方なかったかなとも考えています｡
